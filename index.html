Do Java ao MySQL: Desvendando Nossa API Local de Cadastro de Produtos

Publicado em 23 de Outubro de 2025

Seja bem-vindo(a) ao nosso primeiro mergulho no mundo do desenvolvimento Full Stack! Neste post, não apenas mostrarei o resultado final, mas vou detalhar passo a passo a arquitetura da minha mais recente aplicação. Exploraremos como o poder do Java foi utilizado para construir uma API robusta, como fizemos a mágica da conexão local com o MySQL para garantir a persistência dos dados e, finalmente, como estruturamos os endpoints da API para o crucial processo de cadastro de produtos. Prepare-se para ver como a comunicação entre o back-end e banco de dados acontece na prática!

Quarto passo – Instalar a biblioteca para conexão com o banco de dados MySQL

Iremos utilizar o MySQL Connector/J. Você pode encontrá-lo facilmente pesquisando no Google por “MySQL Connector/J download” ou clicando diretamente no link acima, que levará você à página oficial de download da biblioteca.

O MySQL Connector/J 9.4.0 (ou qualquer versão mais recente) é uma biblioteca (.jar) desenvolvida pela Oracle, responsável por permitir que aplicações Java se conectem e interajam com bancos de dados MySQL. Essa biblioteca atua como um driver JDBC (Java Database Connectivity), sendo essencial para que o Java consiga enviar comandos SQL e receber respostas do banco de forma eficiente.

No Select Operating System a baixo na imagem selecione a opção Plataform Independent

Ao procurar essa biblioteca na internet, você provavelmente encontrará várias versões disponíveis. Não há problema em utilizar uma versão diferente — qualquer uma delas funcionará corretamente. No entanto, para quem utiliza Windows, recomendo baixar a segunda opção (formato .zip), pois é a mais prática para extrair e configurar no Eclipse.

Depois de baixar o arquivo .zip, extraia (descompacte) o conteúdo e abra a pasta. Dentro dela, você verá vários arquivos, mas precisará utilizar apenas um: o mysql-connector-j-9.4.0.jar (ou o nome correspondente à versão que você baixou).

Quinto passo – Montar a classe ConnectionFactory.java

Agora, é hora de criar nossa classe ConnectionFactory, o coração da nossa conexão com o banco de dados MySQL. Compartilharei capturas de tela do código para guiar vocês. No entanto, o aprendizado real virá quando vocês digitarem e experimentarem por conta própria, linha por linha, para dominar a prática!
Quero que vocês prestem bastante atenção nessas variáveis: USER, PASSWORD, DB_NAME, INITIAL_URL e FINAL_URL. São elas que vamos utilizar para realizar a conexão com o banco de dados MySQL.

Na imagem acima, você deve preencher o campo USER com o seu nome de usuário do MySQL (o mesmo que você usa no Workbench) e o campo PASSWORD com a sua senha. Já o INITIAL_URL define o endereço base de conexão com o servidor MySQL, que por padrão será:

jdbc:mysql://localhost:3306/?allowMultiQueries=true&serverTimezone=UTC

Observe que em localhost:3306, o número 3306 representa a porta padrão do MySQL. No entanto, se o seu MySQL estiver configurado em uma porta diferente, basta ajustar esse valor conforme a sua instalação.

Já a variável FINAL_URL é responsável por montar o endereço completo da conexão, unindo o caminho base com o nome do banco de dados definido em DB_NAME. Ela normalmente é declarada assim:

FINAL_URL = "jdbc:mysql://localhost:3306/" + DB_NAME + "?serverTimezone=UTC";

Ou seja, o DB_NAME será concatenado ao final da URL, criando o endereço final do seu banco. Por exemplo, se o nome do seu banco for loja_virtual, a URL resultante será:

jdbc:mysql://localhost:3306/loja_virtual?serverTimezone=UTC

Essa FINAL_URL é a string que o Java usará para estabelecer a conexão real com o banco de dados específico dentro do servidor MySQL.

Continuando ainda na class ConnectionFactory

Vamos estabelecer a conexão com o banco de dados utilizando o método DriverManager.getConnection dentro do método initializeDatabase O método DriverManager.getConnection receberá as variáveis INITIAL_URL, USER, PASSWORD como parâmetros. Uma vez que a conexão (Connection) for obtida, usaremos o método connection.createStatement() para criar um objeto Statement, que é o que nos permite executar comandos SQL.

Uma vez estabelecida a conexão, vamos definir nosso primeiro comando SQL:

String sql = "CREATE DATABASE IF NOT EXISTS " + DB_NAME + ";";

Este comando irá criar um banco de dados cujo nome será o valor contido na variável DB_NAME (no nosso caso, estoque_db, como definimos anteriormente na linha 13).

Após a definição da variável sql (com o comando CREATE DATABASE), vamos redefini-la ou adicionar o comando para usar o banco de dados que acabamos de criar:

sql = "USE " + DB_NAME + ";"; // NOTA: É fundamental executar este comando após a criação do DB.

Depois de apontarmos o banco de dados a ser usado, vamos definir o comando para criar a tabela Produto. Essa tabela terá três colunas (ou campos/atributos): id, nome e preco.

Atenção!!!

Para você entender o que o comando sql está fazendo, é importante você ter pelo menos o básico de comando SQL. Caso você não tenha, é muito importante que você comece a estudar essa parte que será muito importante para o seu aprendizado.

Utilizando a instrução stmt.execute(sql);, vamos executar nosso comando SQL definido na variável sql, através do método execute do objeto Statement.

Caso ocorra algum problema durante a interação ou configuração com o banco de dados, o erro será capturado pela exceção SQLException. No nosso código, utilizaremos essa exceção para disparar uma falha (erro) encapsulada, muitas vezes com o uso de uma RuntimeException (ou uma exceção própria) para interromper o processo de forma controlada.

Caso ocorra um erro de conexão com o banco de dados (por exemplo, credenciais digitadas incorretamente), o método responsável por estabelecer essa conexão irá sinalizar a falha. Essa sinalização é feita pelo uso da instrução throws SQLException, presente na assinatura do método:
public static Connection getConnection() throws SQLException
Isso significa que, se houver um erro, o método lançará (ou propagará) uma exceção do tipo SQLException, que deverá ser tratada pelo código que o chamou.

Quinto e último passo - Montar nossas classes Produto e ProdutoDAO para Inserir um cadastro na nossa tabela Produto

Primeiramente, vamos definir nossa classe Produto. Nela, declararemos os seus atributos (ou campos) e, em seguida, implementaremos os métodos Getters e Setters para manipulação desses dados."

Esta é a nossa classe Produto. Ela servirá como o modelo (ou Model) que utilizaremos na classe ProdutoDAO para realizar a inserção de dados no banco.

Observe que definimos três atributos iniciais: id, nome e preço. Para simplificar o exemplo, manteremos apenas esses campos, mas você está livre para aprimorar e adicionar quantos atributos desejar!

No Java, para acessar ou modificar o valor desses atributos a partir de outras classes, é fundamental implementar os métodos Getters e Setters:

  • O Setter (ou método set) é usado para definir (alterar ou atribuir) o valor de um atributo.
  • O Getter (ou método get) é usado para obter (mostrar ou retornar) o valor contido no atributo.

Bom, agora que montamos a classe Produto, vamos montar a classe ProdutoDAO

import java.sql.PreparedStatement;
Importa a interface PreparedStatement, que é usada para executar comandos SQL de forma segura e eficiente, prevenindo ataques de SQL Injection.

import com.projeto.connection.ConnectionFactory;
Importa a nossa classe ConnectionFactory, criada anteriormente, que é responsável por fornecer a conexão ativa com o banco.

import com.projeto.model.Produto;
Importa a classe Produto, que representa o objeto (modelo de dados) que será manipulado nesta classe DAO.

public class ProdutoDAO {
Declaração da nossa classe ProdutoDAO, que conterá os métodos para realizar as operações de CRUD (Criar, Ler, Atualizar, Deletar).

public void cadastrar(Produto produto) {
Declaração do método cadastrar. Ele é público, não retorna nada (void) e aceita um objeto Produto como parâmetro (o dado a ser inserido).

String sql = "INSERT INTO produto (nome, preco) VALUES (?, ?)";
Define a String SQL para a inserção de dados. O uso de interrogações (?) indica que os valores serão passados posteriormente através do PreparedStatement.

try (Connection conn = ConnectionFactory.getConnection();
Início do bloco try-with-resources. Ele tenta obter uma conexão (conn) usando a ConnectionFactory. O try-with-resources garante que a conexão será fechada automaticamente, mesmo que ocorra um erro.

PreparedStatement stmt = conn.prepareStatement(sql)){
Cria o PreparedStatement (stmt) a partir da conexão (conn), pré-compilando o comando SQL definido.

stmt.setString(1, produto.getNome());
Define o valor do primeiro parâmetro (? na posição 1) da String SQL, usando o nome (`String`) obtido do objeto produto através do seu Getter.

stmt.setDouble(2, produto.getPreco());
Define o valor do segundo parâmetro (? na posição 2) da String SQL, usando o preço (`Double`) obtido do objeto produto.

stmt.executeUpdate();
Executa o comando SQL. O método executeUpdate é usado para comandos que modificam dados (como INSERT, UPDATE, DELETE).

System.out.println("-> [CRUD] Produto cadastrado: " + produto.getNome());
Mensagem de sucesso exibida no console para confirmar que o cadastro foi realizado.

} catch (SQLException e) {
Fim do bloco try e início do bloco catch. Ele captura qualquer SQLException que possa ser lançada durante a conexão ou execução do comando.

System.err.println("-> [ERRO] Falha ao cadastrar produto: " + e.getMessage());
Exibe uma mensagem de erro no console (usando System.err para destaque) caso o cadastro falhe, mostrando a mensagem de erro da exceção (e.getMessage()).

}
Fechamento do bloco catch e, consequentemente, do método cadastrar e da classe ProdutoDAO.
